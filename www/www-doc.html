Computational Billiards WWW interface

Introduction

The web interface allows you to start matches and tournaments, view agent information, and view recorded games. You may either use our web interface at http://altman.stanford.edu/pool or set up your own local database.


Users and Authentication

The web interface uses a very simple authentication scheme. Any web user may sign in by simply logging in as an unknown user. Users can either have regular permissions or be administrators. Administrators have full access to the database and bypass all permission checks. Users, on the other hand, may only create new tournaments and matches, view items (games, matches, etc.) they own, and view completed matches and games where their agents have participated.
If you have installed your own server, it is initialized with an admin account with password admin. This password may be changed using the "List Users" function in the web interface. New administrators may also be assigned using this feature. There is currently no way to delete a user from the web interface.
User passwords are distinct from agent passwords. An agent issuing a register_agent command will create a new entry for the agent if it does not yet exist. The agent's owner will be set based on the owner ID supplied. If no owner ID or an invalid is supplied, the agent will have no owner and will only be accessible by an administrator (which is fine if you are using a local server).
If you are using the supplied client, use the -o command line argument to specify an owner and/or implement the getOwner method in your AI.
User credentials are stored in (unencrypted) cookies. If you need to supply user credentials from a script you may also use the URL parameters username and password.

Viewing Log Files

A simple utility available in the web interface allows any user upload and view a log file (generated by LogWriter) using the JavaScript game viewer. This is accessible from the web interface main page or the upload page.

General information

The web interface closely follows the database design. All selection boxes in forms update the database immediately using AJAX. Text boxes require the use of the submit button.

Running a match

To run a match with specific agents follow the instructions below. We assume here an unprivileged user on our web interface.
  1. Login to the web interface, creating a new user if needed.
  2. Run your agents to register on the system: ./client -o your_userid
  3. Optionally, verify that your agents have been properly registered.
  4. In the main screen, select match under the "Add" header.
  5. Fill in the required information: Name your match (first box), select the participating agents (under the Participants header), and input the number of games left to break for each agent ("Has ___ games left to break").
  6. Noise may be specified using the following notation:
    • 0 = no noise
    • nxa/b/theta/phi/v = specify noise levels, multiplier n is applied to all noise parameters.
    • Any other number is an index to the noise table.
  7. Optionally, you may modify the priority, time model and special rules.
  8. Click "Add" to create the new match.
  9. Start the clients in wait mode (-mw) or fork mode (-mf).
  10. View your match and game results via the List matches option in the main menu.

Running a tournament

A tournament is an automated way to create matches between all pairs of agents of a given set, or between one agent and a set of opponents. Starting a tournament will create the relevant matches.
To create a tournament, follow the above steps for running a match but use add tournament instead of add match, the follow these instructions:
  1. Input a name for your tournament.
  2. If you would like one agent to play against all others (n-1 games) specify a master agent.
  3. Enter the total number of games per match in the "max games" box.
  4. Click "Add"
  5. Under the "Participants" heading select an agent and time and noise information. If using a master agent, you must still add it to the tournament. Then, click "Add".
  6. Repeat until all agents have been added.
  7. Click "Start Tournament" to generate the matches.
  8. Start the clients in wait mode (-mw) or fork mode (-mf).
  9. View your match and game results via the List tournaments option in the main menu.
